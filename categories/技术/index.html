解决 GitHub 访问问题

由于 DNS 污染问题，在大多数运营商网络下 GitHub 访问缓慢，甚至拒绝访问。
对于有限设备的解决方案有以下几种。

一. 更改 DNS 服务器

在 WLAN 设置中更改 DNS 即可。
推荐配置为首选 114.114.114.114 (中国 114 DNS)，备选 8.8.8.8 (谷歌 DNS)。

Hexo 框架搭建简要流程

“Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。”

这是从零开始搭建基于 Hexo 的网页开发环境 (Windows) 的简略步骤。
其他操作系统或更加详细的教程见 Hexo 官方文档 。

网站内容编辑

以 C:/site.sample.io 为例。
对网站的 config 文件进行配置：
先在根目录下的 _config.yml (即C:/site.sample.io/_config.yml) 中填写网站基本信息。
该配置文件会部分覆盖主题的 config，尽量保持两者一致。
在主题目录下按需要编辑 _cofig.yml (即C:/site.sample.ui/themes/sampletheme/_config.yml)。
具体信息见主题使用文档。

配置完成后，即可开始撰写内容。
博客文章为 Markdown 格式文件 (.md)，存储在 C:/site.sample.io/source/_posts 目录下。
需要用到的其他资源、标签页等，均存储在 C:/site.sample.io/source 目录下。
在网页中使用路径时，source 目录即为根目录。
例如，引用一张图片的路径为：
C:/site.sample.io/source/images/sample.jpg
在网页中这样写：
/images/sample.jpg

详细配置过程见 GitHub Pages 官方说明文档。
按要求配置仓库后，即可直接以域名 username.github.io 访问网站。
